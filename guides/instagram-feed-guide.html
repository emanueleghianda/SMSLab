How to create an Instagram-style feed stimulus

This guide explains how to use the SMSLab Instagram Feed Generator to create a realistic, interactive Instagram-style feed stimulus for survey experiments in Qualtrics.

What this tool does The generator creates a complete HTML stimulus that can be copied into Qualtrics. You can add multiple posts, images, carousels, captions, labels, verified badges, comments, call-to-action banners, and interactive icons. If needed, the stimulus can also record participant interactions as Embedded Data.

1. Open the generator

From the SMSLab homepage, click Open Instagram Feed Generator. The generator opens directly in your browser. No installation is required.

The generator has three main areas:

  • Feed setup: general settings for the whole feed.
  • Feed posts: the area where you create and edit each post.
  • Live preview and generated HTML: the area where you check the final feed and copy the code.

2. Configure the feed setup

At the top of the generator, you will find the Feed setup section. This controls the general settings of the feed.

Feed title

The feed title is used internally in the generated HTML file. Participants will normally not see it. You can use a descriptive name such as: 

Instagram feed stimulus - Study 1
Positive feed condition
Negative feed condition
AI disclosure condition

Embedded Data prefix

The Embedded Data prefix determines the name of the variables that Qualtrics will save when participants interact with the feed. The default prefix is: 

igfeed

With this prefix, the generated variables may look like: 

__js_igfeed_post1_like
__js_igfeed_post1_comment
__js_igfeed_post1_save
__js_igfeed_post2_carousel_second_seen

If you create multiple feed versions, it may be useful to change the prefix so that each condition is easy to identify in the dataset. 

positive_feed
negative_feed
ai_feed
human_feed

Interaction tracking

If Enable interaction tracking for Qualtrics is active, the HTML stimulus will try to record participant actions such as likes, comments, saves, direct shares, reposts, carousel views, clicks on AI Information, and clicks on call-to-action banners.

If you only need a visual stimulus and do not care about participant interactions, you can disable tracking. The feed will still look the same, but it will not save click data.

3. Add posts to the feed

Click Add post to create a new post. You can add as many posts as you need. Each post contains two sections:

  • Post setup
  • Labels and other options

An Add post button also appears after each post block. This allows you to insert a new post directly below the current one.

4. Fill in the Post setup section

The Post setup section contains the basic content of the post.

Single-image posts

If the post has only one image, select Single image post and paste one image URL.

Carousel posts

If the post should contain multiple images, select Carousel post and paste one image URL per line. 

https://example.com/image1.jpg
https://example.com/image2.jpg
https://example.com/image3.jpg

The generated post will include carousel arrows, a slide counter, and dots below the image. If interaction tracking is active, Qualtrics can also record whether participants viewed the second image or later.

5. Configure Labels and other options

The Labels and other options section allows you to add visual and interactive elements to each post.

Follow button

The Follow button appears next to the username. If participants click it, the button changes to Following. If tracking is active, the click is saved as Embedded Data.

Verified badge

Use Show verified badge to display a blue verification badge next to the username. This can be useful when manipulating source credibility, account status, or perceived authenticity.

Location label

The location label appears below the username. Examples: 

Milan, Italy
London, United Kingdom
New York, NY

Sponsored label

The sponsored label can be used to simulate advertising disclosure or paid partnership cues. You can customize the text, for example: 

Sponsored
Paid partnership
Advertisement

AI Information label

The AI Information option displays a clickable label under the username. When participants click it, a small pop-up box opens. This is useful for experiments involving AI-generated or AI-assisted content disclosures. 

These images were generated using AI.

Learn more banner

The Learn more banner appears below the image. You can customize the text and optionally add a URL. If a URL is provided, the banner becomes clickable. 

Learn more
Shop now
Discover more
Visit website

Likes and comments count

You can show or hide the number of likes and comments. These fields are useful if social proof is part of your design, but they should be kept constant across conditions unless they are part of the manipulation.

6. Add prewritten comments

For each post, you can add prewritten comments created by the experimenter. Each comment can include:

  • comment username;
  • comment profile image URL;
  • comment text.

In the generated feed, prewritten comments are hidden at first. They appear only when participants click the comment icon for that specific post. This makes the stimulus more realistic and prevents comments from attracting attention before participants interact with the post.

7. Allow participants to comment

When participants click the comment icon, the comment area opens. Participants can type a comment in the Add a comment... field and submit it by pressing Enter or clicking Post.

If tracking is active, the generator can save: 

__js_igfeed_post1_comment_submitted
__js_igfeed_post1_comment_text

This allows you to know whether the participant submitted a comment and what text they entered.

8. Use the live preview

The live preview shows the feed as participants will see it. Use it to check:

  • post order;
  • image loading;
  • carousel behavior;
  • caption formatting;
  • verified badges and labels;
  • AI Information pop-up;
  • prewritten comments;
  • respondent comment field;
  • overall layout.

9. Export the generated feed

When the feed is ready, use one of the export buttons:

  • Copy generated feed HTML: copies the full HTML code to your clipboard.
  • Download feed HTML: downloads the feed as an HTML file.
  • Download this generator: downloads a standalone copy of the generator itself.

10. Insert the feed into Qualtrics

  1. Create a Descriptive Text question.
  2. Open the question editor.
  3. Switch to HTML/source view, often shown as <>.
  4. Paste the full generated HTML code.
  5. Save the question.
  6. Preview the survey and check the feed.
Important Use one generated feed per page/question. Placing multiple generated feeds on the same Qualtrics page may create conflicts between scripts and interaction tracking.

11. Test before collecting data

Before launching your study, complete at least one full test response. Check that:

  • images load correctly;
  • carousel arrows work;
  • icons change color when clicked;
  • comments open only after clicking the comment icon;
  • participants can submit comments;
  • AI Information opens the correct pop-up;
  • Learn more banners work;
  • Embedded Data fields are saved in the dataset.

12. Methodological suggestions

If you use the generator for experimental designs, keep all non-manipulated elements constant across conditions. For example, if you manipulate feed valence, keep constant:

  • number of posts;
  • order of posts;
  • position of the target post;
  • number of images;
  • caption length;
  • number of comments;
  • number of likes;
  • visual quality of images;
  • layout and labels not related to the manipulation.
