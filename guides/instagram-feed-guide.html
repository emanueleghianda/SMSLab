1. What this guide is about

This guide explains how to use HTML-based social media stimuli in Qualtrics using SMSLab.

SMSLab generates realistic, interactive Instagram-style feed stimuli that can be copied directly into Qualtrics. Unlike static images or screenshots, HTML stimuli allow participants to interact with the stimulus: they can click like, open comments, write comments, save posts, open AI Information labels, view carousel images, or click a call-to-action banner.

The goal is to help researchers create social-media-style stimuli that are more realistic, flexible, and suitable for experimental research.

2. Why use HTML stimuli instead of static images?

Many survey experiments use static screenshots of social media posts. This is simple and often effective, but it has some limitations.

A static image can show what a post looks like, but it cannot reproduce how people interact with social media content. Participants cannot click icons, open comments, scroll through a carousel, or submit a comment.

HTML stimuli are useful when the interaction itself matters.

For example, HTML stimuli can include:

                                clickable like icons
                                clickable comment icons
                                clickable repost/direct/save icons
                                carousel navigation
                                hidden comments that open when clicked
                                AI Information pop-ups
                                Learn more / Shop now banners
                                participant-written comments
                                interaction tracking through Qualtrics Embedded Data

This makes HTML stimuli especially useful for studies on social media advertising, influencer marketing, AI-generated content disclosure, sponsored content, brand communication, online engagement, comments, social proof, and platform interface cues.

3. When should you use HTML stimuli?

Use HTML stimuli when your study benefits from realism, interaction, or behavioral tracking.

For example, HTML stimuli are useful if you want to know whether participants:

                                liked a post
                                opened the comment section
                                submitted a comment
                                saved a post
                                clicked a call-to-action banner
                                opened an AI disclosure label
                                viewed additional carousel images

If your study only requires participants to look at a post and answer questions, a static image may be enough. The advantage of HTML appears when you want the stimulus to behave more like a real social media interface.

4. What SMSLab generates

SMSLab generates a complete HTML stimulus. The output includes:

                                HTML structure
                                CSS styling
                                icons
                                interactive JavaScript
                                Qualtrics-compatible tracking code

The generated feed can include multiple posts. Each post can be customized with:

                                username
                                profile image
                                verified badge
                                timestamp
                                location label
                                sponsored label
                                AI Information label
                                single image or carousel
                                caption
                                likes count
                                comments count
                                prewritten comments
                                respondent comment field
                                Learn more banner
                                interactive icons

Once generated, the HTML can be copied into a Qualtrics Descriptive Text question.

5. Basic workflow

The general workflow is:

                                1. Create the feed in SMSLab.
                                2. Copy or download the generated HTML.
                                3. Create a Descriptive Text question in Qualtrics.
                                4. Paste the generated HTML into the HTML/source view of the question.
                                5. Add the required Embedded Data fields in Survey Flow.
                                6. Add separate Embedded Data fields for the experimental condition.
                                7. Preview and test the survey.
                                8. Launch data collection.

6. Feed setup in SMSLab

The first section of the generator is Feed setup.

This section includes:

                                Feed title
                                Embedded Data prefix
                                Interaction tracking options

Feed title

The Feed title is used internally in the generated HTML. Participants usually do not see it.

Examples:

                                Instagram feed stimulus - Study 1
                                Positive feed condition
                                AI disclosure feed
                                Sponsored content condition

Embedded Data prefix

The Embedded Data prefix is used to name the variables that Qualtrics will save when participants interact with the feed.

For example, if the prefix is:

                                igfeed

the interaction fields may look like:

                                __js_igfeed_post1_like
                                __js_igfeed_post1_comment
                                __js_igfeed_post1_save
                                __js_igfeed_post2_carousel_second_seen

The prefix is only a naming tool. It helps organize the interaction variables created by the stimulus.

It is important not to confuse the Embedded Data prefix with the experimental condition.

7. Prefix vs. experimental condition

The Embedded Data prefix does not identify the experimental condition.

You should think of two separate types of Embedded Data:

                                1. Condition Embedded Data
                                2. Interaction Embedded Data

Condition Embedded Data tells you which condition the participant was assigned to.

Interaction Embedded Data tells you what the participant did inside the feed.

For example, imagine a 2 × 2 experiment:

                                AI disclosure: AI vs No AI
                                Engagement level: High vs Low

In Qualtrics, you should create condition variables such as:

                                ai_disclosure = AI
                                engagement_level = High
                                condition = AI_HighEngagement

These variables identify the experimental condition.

Separately, SMSLab can generate interaction variables such as:

                                __js_igfeed_post1_like
                                __js_igfeed_post1_comment
                                __js_igfeed_post1_save
                                __js_igfeed_post1_carousel_second_seen

These variables identify participant behavior inside the feed.

A dataset may therefore contain both:

                                condition = AI_HighEngagement
                                ai_disclosure = AI
                                engagement_level = High
                                __js_igfeed_post1_like = 1
                                __js_igfeed_post1_comment = 0
                                __js_igfeed_post1_save = 1

The condition variables and the interaction variables serve different purposes.

8. Should you change the prefix across conditions?

In most experiments, no.

If each participant sees only one feed, it is usually better to keep the same prefix across all experimental conditions.

For example, use the same prefix:

                                igfeed

for all conditions.

This means that every participant will have the same interaction columns in the dataset:

                                __js_igfeed_post1_like
                                __js_igfeed_post1_comment
                                __js_igfeed_post1_save

Then you analyze these interaction variables using your separate condition variables:

                                condition
                                ai_disclosure
                                engagement_level

This is usually cleaner than changing the prefix for each condition.

Recommended approach

For a between-subjects experiment where each participant sees only one feed:

                                Keep the same prefix across conditions.
                                Use separate Embedded Data fields to store the condition.

Example:

                                Prefix: igfeed
                                Condition Embedded Data:
                                condition = AI_HighEngagement
                                ai_disclosure = AI
                                engagement_level = High
                                Interaction Embedded Data:
                                __js_igfeed_post1_like
                                __js_igfeed_post1_comment
                                __js_igfeed_post1_save

When should you change the prefix?

Change the prefix only when the same participant sees multiple different feeds in the same survey and you need to store interactions separately.

Example:

                                Prefix for first feed: feed1
                                Prefix for second feed: feed2

This would produce:

                                __js_feed1_post1_like
                                __js_feed1_post1_save
                                __js_feed2_post1_like
                                __js_feed2_post1_save

This is useful if one participant interacts with more than one feed and you want to know which interaction belongs to which feed.

9. Adding posts to the feed

Click Add post to add a new post.

Each post is divided into two sections:

                                Post setup
                                Labels and other options

You can add as many posts as needed.

For experimental research, it is usually recommended to keep the number of posts constant across conditions unless the number of posts is part of the manipulation.

For example, if Condition A contains five posts, Condition B should usually also contain five posts.

10. Post setup

The Post setup section contains the main content of each post.

Username

The account name displayed at the top of the post.

Example:

                                urbanstyle_daily
                                travelnotes
                                brandname

Timestamp

The time/date label associated with the post.

Examples:

                                2 hours ago
                                Yesterday
                                May 2026

You can leave it empty if time is not relevant to the stimulus.

Profile image URL

The URL of the profile image shown next to the username.

The image must be publicly accessible online.

Example:

                                https://example.com/profile-image.jpg

Post format

You can choose between:

                                Single image post
                                Carousel post

Use Single image post when the post contains one image.

Use Carousel post when the post contains multiple images.

Image URL(s)

For a single-image post, paste one image URL.

For a carousel post, paste one image URL per line.

Example:

                                https://example.com/image1.jpg
                                https://example.com/image2.jpg
                                https://example.com/image3.jpg

In the generated feed, carousel posts include navigation arrows, a slide counter, and carousel dots.

Caption

The text displayed below the post.

Example:

                                A new way to experience urban style.
                                #Fashion #Design #NewCollection

Hashtags are automatically styled in blue.

11. Labels and other options

The Labels and other options section allows you to add visual and interactive elements to each post.

Follow button

If enabled, the post displays a Follow button next to the username.

When participants click it, it changes to:

                                Following

If tracking is active, this interaction can be saved.

Verified badge

If enabled, a blue verified badge appears next to the username.

This can be useful for studies involving:

                                source credibility
                                account status
                                brand legitimacy
                                influencer authenticity
                                perceived trustworthiness

Location label

The location label appears below the username.

Examples:

                                Milan, Italy
                                London, United Kingdom
                                New York, NY

Sponsored label

The sponsored label can simulate advertising disclosure.

Examples:

                                Sponsored
                                Advertisement
                                Paid partnership

If sponsored disclosure is not part of the manipulation, keep it constant across conditions.

AI Information label

The AI Information option adds a clickable label under the username.

When participants click it, a pop-up opens.

Example message:

                                These images were generated using AI.

This is useful for studies about AI-generated content, AI disclosure, authenticity, synthetic media, or algorithmic transparency.

Learn more banner

The Learn more banner appears below the image and above the interaction icons.

You can customize the text:

                                Learn more
                                Shop now
                                Discover more
                                Visit website

You can also add a URL. If a URL is included, the banner becomes clickable.

If tracking is active, SMSLab can save whether the participant clicked the banner.

Likes and comments count

You can display the number of likes and comments.

Examples:

                                1,248 likes
                                View all 36 comments

These are social proof cues. Keep them constant across conditions unless they are part of the manipulation.

12. Prewritten comments

For each post, you can add prewritten comments created by the experimenter.

Each prewritten comment can include:

                                comment username
                                comment profile image URL
                                comment text

Example:

                                Username: user_01
                                Profile image URL: https://example.com/profile.jpg
                                Comment text: Love this post!

In the generated feed, prewritten comments are hidden by default. They appear only when the participant clicks the comment icon for that specific post.

This makes the stimulus more realistic and prevents comments from attracting attention before participants choose to open them.

13. Respondent comments

Participants can also write their own comments.

When participants click the comment icon, the comment area opens. They can type a comment into:

                                Add a comment...

and submit it by clicking:

                                Post

or pressing Enter.

If interaction tracking is active, Qualtrics can save:

                                __js_igfeed_post1_comment_submitted
                                __js_igfeed_post1_comment_text

This can be useful if written comments are part of the dependent variable, a behavioral indicator, or an exploratory measure.

14. Using the live preview

The live preview shows the feed as participants will see it.

Use the preview to check:

                                post order
                                image loading
                                caption formatting
                                verified badges
                                sponsored labels
                                AI Information pop-ups
                                Learn more banners
                                carousel navigation
                                prewritten comments
                                respondent comment area
                                icon behavior
                                overall layout

The preview helps identify errors before copying the generated HTML into Qualtrics.

15. Exporting the HTML

When the feed is ready, use one of the export buttons.

Copy generated feed HTML copies the full HTML code to your clipboard.

Download feed HTML downloads the generated stimulus as an HTML file.

The generated HTML includes:

                                layout
                                style
                                images
                                icons
                                interaction logic
                                comment behavior
                                Qualtrics tracking script
Do not copy only part of the code. The full generated HTML is required.

16. Inserting the stimulus into Qualtrics

In Qualtrics:

                                1. Create or open your survey.
                                2. Create a new block for the stimulus or condition.
                                3. Add a Descriptive Text question.
                                4. Open the Rich Content Editor.
                                5. Switch to HTML/source view, often shown as <>.
                                6. Paste the full generated HTML.
                                7. Save the question.
                                8. Preview the survey.

The stimulus should usually appear before manipulation checks, mediators, dependent variables, or other post-exposure measures.

17. Setting up interaction Embedded Data

If interaction tracking is enabled, SMSLab shows a list of Embedded Data fields.

Examples:

                                __js_igfeed_post1_like
                                __js_igfeed_post1_comment
                                __js_igfeed_post1_comment_submitted
                                __js_igfeed_post1_comment_text
                                __js_igfeed_post1_repost
                                __js_igfeed_post1_direct
                                __js_igfeed_post1_save
                                __js_igfeed_post1_carousel_slide
                                __js_igfeed_post1_carousel_second_seen
                                __js_igfeed_post1_ai_info_opened
                                __js_igfeed_post1_cta_click

In Qualtrics, add these fields in Survey Flow.

Step-by-step:

                                1. Go to Survey Flow.
                                2. Click Add a New Element Here.
                                3. Select Embedded Data.
                                4. Add every field shown by SMSLab.
                                5. Leave the values blank.
                                6. Move the Embedded Data element above the block containing the stimulus.
                                7. Save the Survey Flow.

The field names must match exactly.

18. Setting up condition Embedded Data

Interaction Embedded Data is not enough to identify the experimental condition.

You should also add separate condition variables.

For example, in a 2 × 2 experiment:

                                AI disclosure: AI vs No AI
                                Engagement level: High vs Low

you may create condition fields such as:

                                condition
                                ai_disclosure
                                engagement_level

For a participant assigned to the AI + High Engagement condition, the values could be:

                                condition = AI_HighEngagement
                                ai_disclosure = AI
                                engagement_level = High

For a participant assigned to the No AI + Low Engagement condition, the values could be:

                                condition = NoAI_LowEngagement
                                ai_disclosure = NoAI
                                engagement_level = Low

This allows you to analyze the effect of the manipulated factors while keeping interaction variables consistent across participants.

19. Example dataset structure

A clean dataset may contain both condition variables and interaction variables.

Example:

                                condition = AI_HighEngagement
                                ai_disclosure = AI
                                engagement_level = High
                                __js_igfeed_post1_like = 1
                                __js_igfeed_post1_comment = 1
                                __js_igfeed_post1_comment_submitted = 0
                                __js_igfeed_post1_save = 1
                                __js_igfeed_post1_ai_info_opened = 1

The condition variables tell you what the participant saw.

The interaction variables tell you what the participant did.

20. Testing before data collection

Always complete a full test before launching data collection.
Recommended test procedure:
                                1. Open the Qualtrics survey preview.
                                2. View the feed stimulus.
                                3. Click like, comment, repost, direct, save, and follow.
                                4. Open the AI Information box if present.
                                5. Click the Learn more banner if present.
                                6. Scroll through carousel images if present.
                                7. Submit a written comment.
                                8. Complete the survey.
                                9. Open the recorded response.
                                10. Check whether Embedded Data fields contain the expected values.

Example expected result:

                                condition = AI_HighEngagement
                                ai_disclosure = AI
                                engagement_level = High
                                __js_igfeed_post1_like = 1
                                __js_igfeed_post1_comment = 1
                                __js_igfeed_post1_comment_submitted = 1
                                __js_igfeed_post1_comment_text = "Interesting post!"
                                __js_igfeed_post1_save = 1
                                __js_igfeed_post2_carousel_second_seen = 1

21. Common problems and solutions

The stimulus does not appear correctly

Check that the full HTML code was pasted into HTML/source view, not into the normal visual editor.

Also check that all image URLs are public and accessible.

The images do not load

Make sure the image link points directly to an image file. Some links point to webpages rather than image files.

The interaction fields are blank

Check that:

                                the fields were added in Survey Flow
                                the fields were placed before the stimulus block
                                the field names match exactly
                                the __js_ prefix is included when required
                                the participant actually interacted with the stimulus
                                a full test response was completed

The condition variable is missing

Interaction tracking does not automatically create condition variables.

You need to create condition Embedded Data separately in Survey Flow.

The prefix is different across conditions

This is not necessarily wrong, but it is often unnecessary.

If each participant sees only one feed, use the same prefix across conditions and store the condition separately.

22. Methodological recommendations

For experimental research, keep all non-manipulated elements constant across conditions.

If a feature is not part of the manipulation, keep it the same.

Usually keep constant:

                                number of posts
                                position of the target post
                                number of images
                                caption length
                                profile image style
                                number of comments
                                number of likes
                                presence of location labels
                                presence of sponsored labels
                                presence of verified badges
                                visual quality of images
                                layout and spacing
                                Embedded Data prefix

Only change the elements that are part of the experimental manipulation.

For example, if the manipulation is AI disclosure, the AI Information label should change across conditions, while the rest of the feed should remain as similar as possible.

23. Suggested reporting in a methods section

You can describe the stimulus creation process like this:

                                Experimental stimuli were created using SMSLab, an open-source tool for generating realistic social-media-style HTML stimuli for Qualtrics. The stimulus consisted of an Instagram-style feed containing multiple posts. The target post was embedded within the feed and included interactive interface elements such as like, comment, save, and carousel navigation. The HTML stimulus was inserted into Qualtrics using a Descriptive Text question.

If you used tracking, you can add:

                                The study also recorded behavioral interaction indicators through Qualtrics Embedded Data, including whether participants liked the post, opened the comment section, saved the post, viewed additional carousel images, clicked the call-to-action banner, opened the AI Information label, or submitted a written comment.

If you used experimental condition fields, you can add:

                                Experimental condition assignment was stored separately using Qualtrics Embedded Data fields, allowing interaction variables to be analyzed as behavioral responses within each condition.

24. Final checklist

Before launching, check that:

                                the feed looks realistic
                                all images load correctly
                                all posts are in the correct order
                                the target post is in the intended position
                                the manipulation is visible but not too obvious
                                non-manipulated features are constant across conditions
                                the Embedded Data prefix is consistent across conditions when appropriate
                                condition variables are created separately
                                the generated HTML is pasted correctly into Qualtrics
                                interaction Embedded Data fields are declared in Survey Flow
                                the survey has been previewed and tested
                                a test response shows the expected Embedded Data values

This workflow helps ensure that the stimulus is visually realistic, technically reliable, and methodologically controlled.
