Randomization and Embedded Data in Qualtrics

This guide explains how to randomize experimental conditions in Qualtrics and how to save participant interactions with SMSLab stimuli using Embedded Data.

What you will learn You will learn how to set up one condition per block, randomize participants across conditions, declare Embedded Data fields, and verify that interaction data are correctly saved in your dataset.

1. Why Embedded Data matters

SMSLab stimuli can record interactions such as likes, comments, reposts, direct shares, saves, carousel views, clicks on AI Information, clicks on Learn more banners, and comments submitted by respondents.

In Qualtrics, these values are stored as Embedded Data. Embedded Data fields behave like variables in your final dataset. They do not need to be shown to participants.

2. Understand the field names

The generator displays a list of fields that should be added to Qualtrics Survey Flow. Examples: 

__js_igfeed_post1_like
__js_igfeed_post1_comment
__js_igfeed_post1_comment_submitted
__js_igfeed_post1_comment_text
__js_igfeed_post1_repost
__js_igfeed_post1_direct
__js_igfeed_post1_save
__js_igfeed_post1_carousel_slide
__js_igfeed_post1_carousel_second_seen
__js_igfeed_post1_ai_info_opened

These names are structured as follows: 

__js_ + prefix + post number + interaction
PartMeaning
__js_Prefix used by newer Qualtrics JavaScript Embedded Data behavior.
igfeedThe default Embedded Data prefix from the generator.
post1The post number in the generated feed.
likeThe specific interaction being tracked.

3. Create one block per condition

If you are running an experiment with multiple conditions, create one Qualtrics block for each condition.

Example for a two-condition study: 

Block 1: Positive Instagram feed
Block 2: Negative Instagram feed

Example for a 2 × 2 design: 

Block 1: AI disclosure + High engagement
Block 2: AI disclosure + Low engagement
Block 3: No AI disclosure + High engagement
Block 4: No AI disclosure + Low engagement

Inside each block, add a Descriptive Text question containing the HTML stimulus generated by SMSLab.

4. Add the generated HTML to Qualtrics

  1. Open the relevant condition block.
  2. Add a Descriptive Text question.
  3. Open the Rich Content Editor.
  4. Switch to HTML/source view, often shown as <>.
  5. Paste the full generated HTML code.
  6. Save the question.
Tip Do not paste only part of the HTML. The generated code includes the layout, CSS, icons, and JavaScript needed for the stimulus to work.

5. Add Embedded Data fields in Survey Flow

Before collecting data, you need to declare the Embedded Data fields shown by the generator.

  1. Open Survey Flow.
  2. Click Add a New Element Here.
  3. Select Embedded Data.
  4. Add all the field names listed by the generator.
  5. Leave the values blank.
  6. Move this Embedded Data element above the randomized experimental blocks.
  7. Save the Survey Flow.

For example, if your stimulus tracks likes, saves, and comments for Post 1, add fields such as: 

__js_igfeed_post1_like
__js_igfeed_post1_save
__js_igfeed_post1_comment
__js_igfeed_post1_comment_submitted
__js_igfeed_post1_comment_text

6. Randomize participants across conditions

In Survey Flow, add a Randomizer element. Place your condition blocks inside the Randomizer and configure it to show one element.

A typical structure looks like this: 

Survey Flow
│
├── Embedded Data fields
│
├── Randomizer: randomly present 1 of the following elements
│   ├── Block: Condition 1
│   ├── Block: Condition 2
│   ├── Block: Condition 3
│   └── Block: Condition 4
│
└── Outcome measures / questionnaire blocks

This ensures that each participant sees only one feed condition.

7. Save the experimental condition

It is good practice to save the assigned condition as an Embedded Data variable. For example: 

condition = positive_feed
condition = negative_feed

To do this, add an Embedded Data element inside or immediately before each condition block and set a field such as: 

condition = positive_feed

This makes analysis easier because your dataset will contain an explicit condition variable.

9. Test before launching

Always test your survey before collecting data.

  1. Preview the survey.
  2. Interact with the stimulus: like, save, comment, open AI Information, click Learn more, scroll the carousel.
  3. Complete the survey.
  4. Open the response data.
  5. Check whether the Embedded Data fields contain the expected values.

Example expected output: 

__js_igfeed_post1_like = 1
__js_igfeed_post1_comment = 1
__js_igfeed_post1_comment_submitted = 1
__js_igfeed_post1_comment_text = "Interesting post!"
__js_igfeed_post2_carousel_second_seen = 1

10. Common problems and fixes

The Embedded Data fields are blank

  • Check that the fields were added in Survey Flow.
  • Check that the Embedded Data element appears before the stimulus block.
  • Check that the field names match exactly, including the __js_ prefix.
  • Preview the survey and complete a full test response.

The stimulus does not look correct in Qualtrics

  • Make sure you pasted the full HTML code.
  • Make sure you used HTML/source view, not the normal visual editor.
  • Check whether external image URLs are working.

The condition variable is missing

  • Add a separate Embedded Data field called condition.
  • Set its value differently inside each condition block.

11. Recommended workflow

  1. Create each feed condition in SMSLab.
  2. Use a different Embedded Data prefix for each condition if needed.
  3. Copy each generated HTML stimulus into its own Qualtrics block.
  4. Add all Embedded Data fields in Survey Flow.
  5. Add a condition variable.
  6. Randomize condition blocks.
  7. Test the full survey before launching.
